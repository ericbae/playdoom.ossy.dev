How it works

Your copy runs on your account.

01

Deploy

Cloudflare clones the template and provisions the Worker, Static Assets, and Durable Object room router.

02

Host

Open your Worker URL, choose a FreeDM map and time limit, then copy the invite link.

03

Play

Friends join before the match starts. The browser engine handles the deathmatch session.

Step-by-step

Deploy without granting access to every repo.

Create one empty GitHub repository first, then authorize Cloudflare only for that repository during setup. Cloudflare will copy the PlayDoom template into it and deploy your Worker.

  1. 01

    Create an empty GitHub repo

    Use your own GitHub account. Do not add a README, license, or gitignore file.

  2. 02

    Click Deploy to Cloudflare

    Cloudflare opens the setup flow for the public playdoom-template source repo.

  3. 03

    Connect your Git account

    When GitHub asks what Cloudflare can access, choose only the empty PlayDoom repo you created.

  4. 04

    Set up the application

    Use the empty repo as the destination, choose a Worker name, and keep the generated Cloudflare resources.

  5. 05

    Add DOOM_KEY

    Paste a long random value when Cloudflare asks for the DOOM_KEY secret. This signs private room links.

  6. 06

    Open your Worker URL

    Cloudflare deploys your copy and shows a workers.dev URL. Open it, start a room, and share the invite link.

Included

FreeDM maps, browser engine, room links.

The template bundles generated map previews, FreeDM assets, and a WebAssembly Doom-compatible engine behind a simple browser interface.

Current limits

Built for small private games.

  • Up to 4 playable slots.
  • Players should join before the host starts the match.
  • Frags are shown in-game, but there is no persistent scoreboard yet.
  • The deploy uses FreeDM/Freedoom assets, not commercial game data.

Template repo

Deploy or fork the Worker template.

The template contains the Worker, Durable Object room router, browser UI, asset preparation scripts, smoke tests, and self-hosting docs.
